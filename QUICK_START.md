# 🚀 快速開始指南

## 新功能快速體驗

### 📊 功能一：資料統計儀表板

#### 🎯 3 步驟體驗統計儀表板

1. **訪問儀表板**
   ```
   http://localhost:5000/dashboard
   ```

2. **查看統計資料**
   - 📁 已生成檔案數
   - 👥 病人總數
   - 🏥 就診記錄數
   - 🩺 疾病、觀察、藥物統計

3. **查看圖表**
   - 🥧 圓餅圖：資源類型分布
   - 📊 長條圖：資源數量統計
   - 📋 最近生成檔案列表

---

### 🎯 功能二：情境預設模式

#### 🎯 5 步驟快速生成糖尿病患者資料

1. **訪問首頁**
   ```
   http://localhost:5000
   ```

2. **選擇情境**
   - 在「情境預設模式」下拉選單
   - 選擇「🩺 糖尿病患者」

3. **自動填入參數**
   - 系統顯示確認訊息
   - 自動填入建議數值：
     * 病人數量：5
     * 疾病：1 種
     * 觀察：5 項
     * 藥物：3 種
     * 就診：2 次

4. **開始生成**
   - 點擊「開始生成」按鈕
   - 查看進度條

5. **查看結果**
   - 生成完成後顯示統計
   - 下載生成的檔案

## 💻 完整使用流程

### 情境 1：新手快速上手

```
1. 打開瀏覽器 → http://localhost:5000
2. 選擇「🏥 健康檢查」情境
3. 確認自動填入的參數
4. 選擇「僅生成檔案，不上傳」
5. 點擊「開始生成」
6. 等待 10-20 秒
7. 下載生成的 JSON 檔案
8. 到儀表板查看統計 → /dashboard
```

### 情境 2：測試 FHIR 伺服器

```
1. 打開首頁
2. 選擇「🩺 糖尿病患者」情境
3. 調整病人數量為 2
4. 選擇上傳選項：
   - 國際 HAPI 伺服器（推薦）
   - 或台灣 TWCORE 伺服器
5. 開始生成並上傳
6. 查看上傳結果統計
```

### 情境 3：大量測試資料生成

```
1. 選擇「👴 老年人綜合照護」
2. 將病人數量改為 20
3. 生成複雜的多重慢性病資料
4. 查看儀表板統計結果
```

## 💡 使用技巧

### 技巧 1：組合不同情境

```
生成老年病房的測試資料：
1. 先用「糖尿病患者」生成 5 人

```

```
1. 選擇「糖尿病患者」
2. 系統填入 5 個病人
3. 手動改為 10 個病人

```

```
開發過程中：
1. 每次生成後查看儀表板
2. 確保測試資料涵蓋各種情境
3. 避免資料過於單一
```

## 📊 效能指標

### 生成速度參考

| 病人數 | 預估時間 | 檔案大小 |
|--------|---------|----------|
| 1 人   | ~1 秒   | ~15 KB   |
| 5 人   | ~5 秒   | ~75 KB   |
| 10 人  | ~10 秒  | ~150 KB  |
| 50 人  | ~50 秒  | ~750 KB  |
| 100 人 | ~2 分鐘 | ~1.5 MB  |

### 上傳速度參考

取決於網路和伺服器：
- 本地伺服器：非常快
- HAPI 公共伺服器：中等（可能有限流）
- TWCORE 伺服器：視網路狀況

## ⚠️ 注意事項

### 上傳相關

1. **TWCORE 伺服器**
   - 如遇超時，建議先「僅生成檔案」
   - 可以稍後手動上傳

2. **HAPI 伺服器**
   - 建議用於測試
   - 資料會被公開訪問
   - 定期會清理資料

3. **自訂伺服器**
   - 確保伺服器可連接
   - 檢查認證設定

### 檔案管理

1. **定期清理**
   - `output/` 目錄會持續累積檔案
   - 建議定期手動清理舊檔案

2. **Docker Volume**
   - 檔案保存在 Docker volume 中
   - 刪除容器前記得備份重要檔案
